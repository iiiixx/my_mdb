Система рекомендаций фильмов

Программный комплекс для формирования персонализированных рекомендаций фильмов на основе анализа пользовательских рейтингов.

Система использует модель матричной факторизации (SVD), которая позволяет предсказывать предпочтения пользователей и формировать рекомендации на основе скрытых (латентных) факторов пользователей и фильмов.

Архитектура системы разделена на несколько компонентов:

Go backend, отвечающий за обработку запросов и управление данными

Python микросервис рекомендаций, выполняющий вычисления на основе обученной модели

база данных PostgreSQL, хранящая информацию о фильмах и рейтингах пользователей.

Проект демонстрирует полный цикл работы рекомендательной системы:
подготовку данных, обучение модели и выдачу рекомендаций в режиме реального времени.

Основные возможности

персонализированные рекомендации фильмов

выставление пользовательских рейтингов

отображение популярных фильмов

подборки фильмов по жанрам

микросервисная архитектура с использованием gRPC

Архитектура системы

Система состоит из двух основных сервисов.

Backend сервис (Go)

Backend реализован на языке Go и выполняет следующие функции:

обработка HTTP-запросов пользователей

управление пользовательскими рейтингами

выборка данных о фильмах из базы данных

формирование подборок фильмов

взаимодействие с сервисом рекомендаций через gRPC

Технологии:

Go

PostgreSQL

gRPC client

HTML шаблоны

OMDb API для получения постеров и дополнительной информации о фильмах

Сервис рекомендаций (Python)

Отдельный микросервис, реализованный на Python, отвечает за вычисление рекомендаций.

Он выполняет:

загрузку обученной модели

вычисление персонализированных рекомендаций

поиск похожих фильмов

обработку запросов через gRPC API

Алгоритм рекомендаций

Для построения рекомендаций используется модель сингулярного разложения матрицы предпочтений пользователей (SVD).

Модель предсказывает рейтинг пользователя для фильма по формуле:

r̂_ui = μ + b_u + b_i + q_i^T p_u

где:

μ — глобальное среднее значение рейтинга

b_u — смещение пользователя

b_i — смещение фильма

p_u — латентный вектор пользователя

q_i — латентный вектор фильма

Рекомендации формируются на основе фильмов с наибольшим предсказанным рейтингом.

Поиск похожих фильмов

Похожие фильмы определяются на основе сходства латентных векторов фильмов, полученных при обучении модели.

Для оценки сходства используется косинусная мера:

sim(i,j) = (q_i · q_j) / (||q_i|| ||q_j||)
Поток данных системы

Работа системы разделяется на два этапа.

1. Оффлайн этап (обучение модели)
Датасет рейтингов
        ↓
Предобработка данных
        ↓
Формирование матрицы пользователь–фильм
        ↓
Обучение модели SVD
        ↓
Сохранение параметров модели
2. Онлайн этап (выдача рекомендаций)
Пользовательский запрос
        ↓
Go backend
        ↓
Запрос рекомендаций через gRPC
        ↓
Python сервис рекомендаций
        ↓
Получение списка фильмов
        ↓
Формирование ответа API
